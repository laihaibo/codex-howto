# finishing-a-development-branch

完成开发后指导如何集成工作的技能。

## 触发条件

```
> 这个分支的开发完成了，接下来怎么做？
> $finishing-branch 所有测试通过了
```

## 集成决策流程

```
开发完成
    ↓
测试全部通过？
    ↓ 是
需要合入主干？
    ├── 是 → Pull Request
    └── 否 → 保持在分支上
        ↓
    保持独立发布？
        ├── 是 → 直接部署分支
        └── 否 → 暂存
```

## 集成方式

| 方式 | 适用场景 |
|------|---------|
| Pull Request | 需要审查的合入 |
| 直接 merge | 紧急修复或独立工作 |
| 保持在分支 | 功能开关控制的发布 |

## 完成清单

- [ ] 所有测试通过
- [ ] 代码审查完成（如适用）
- [ ] 文档更新
- [ ] 变更日志更新

# verification-before-completion

完成工作前验证结果的技能。

## 核心原则

**证据 > 声明** —— 在声称工作完成前，必须有证据证明。

## 验证步骤

```
完成编码 → 运行测试 → 显示输出 → 确认通过
                                   ↓
                            标记完成
```

## 验证方法

| 工作类型 | 验证方式 |
|---------|---------|
| 功能实现 | 运行测试 + 手动验证 |
| Bug 修复 | 触发原问题场景确认修复 |
| 性能优化 | 对比优化前后指标 |
| 重构 | 确保所有测试通过 |

## 常见误区

- ❌ "感觉应该能工作"
- ❌ "类似问题之前修复过"
- ✅ "这是测试输出，全部通过"
- ✅ "这是实际的浏览器截图"

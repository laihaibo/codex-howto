# requesting-code-review

请求代码审查技能，在重要变更完成后请求反馈。

## 触发条件

```
> 完成这个功能后请求代码审查
> $review 合并前审查我的代码
```

## 适用场景

- 完成主要功能的实现
- 修复复杂 bug 后
- 合并 PR 前
- 改动影响多个模块时

## 审查要点

| 类别 | 检查项 |
|------|--------|
| 正确性 | 功能是否按预期工作 |
| 可维护性 | 代码是否易于理解和修改 |
| 安全性 | 是否存在安全漏洞 |
| 测试 | 是否有充分的测试覆盖 |

## Codex 会

- 生成变化的摘要
- 突出潜在风险区域
- 按严重性分类问题
- 提出改进建议

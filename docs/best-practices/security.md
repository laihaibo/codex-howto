# 安全实践

## 理解沙箱模型

Codex 使用沙箱保护你的系统和数据安全：

| 模式 | 文件读取 | 文件写入 | 命令执行 |
|------|---------|---------|---------|
| 沙箱模式 | ⚠️ 限制 | ⚠️ 限制 | ⚠️ 限制 |
| 信任模式 | ✅ 全权限 | ✅ 全权限 | ⚠️ 需确认 |

## 信任级别

### 何时信任

仅在以下情况标记项目为信任：

- 你自己的项目
- 可信的内部代码库
- 已验证的开源项目

### 何时不信任

- 审查未知代码
- 处理用户提供的代码
- 在共享环境工作

## 安全使用清单

- [ ] 所有 API 密钥通过环境变量配置
- [ ] 不在对话中粘贴敏感信息
- [ ] 启用 Secret Scanning
- [ ] 定期运行安全扫描
- [ ] 审查 AI 生成的外部请求代码

## Secret Scanning

Codex 会自动扫描提交中的密钥和令牌，防止泄露：

> 扫描最近 5 次提交中是否包含密钥

## 安全扫描能力

使用 Codex Security 插件进行全面扫描：

| 扫描类型 | 触发方式 |
|---------|---------|
| 仓库安全扫描 | "扫描安全问题" |
| 深度安全扫描 | "深度安全分析" |
| 变更安全审查 | "检查这次变更的安全性" |

# systematic-debugging

系统化调试技能，在提出修复前先定位和分析问题根因。

## 触发条件

```
> 这个测试为什么失败？系统化调试一下
> $debugging "登录时返回 500 错误"
> 这个 bug 的原因是什么？
```

## 调试流程

1. **复现问题**：确认能否稳定复现
2. **收集证据**：错误日志、堆栈、输入输出
3. **形成假设**：列出可能的原因
4. **验证假设**：通过测试排除不可能原因
5. **定位根因**：找到真正的问题来源
6. **实施修复**：针对根因而非症状

## 调试技巧

- 二分法定位（注释掉一半代码看问题是否消失）
- 添加日志追踪数据流
- 检查最近修改（git blame/log）
- 比较工作与不工作的差异

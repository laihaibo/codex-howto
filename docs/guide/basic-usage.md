# 基本使用

## 启动 Codex

### 交互模式

最简单的使用方式是启动交互会话：

```bash
cd /path/to/your/project
codex
```

交互模式下，你可以直接与 AI 对话，它会根据你的指令操作项目文件。

### 单次任务模式

执行单个任务后退出：

```bash
codex "创建一个 React 组件用于显示用户头像"
```

### 继续会话

恢复上一次的对话上下文：

```bash
codex --continue
```

## 输入提示

### 基本对话

输入你的需求，Codex 会理解并执行：

```
> 创建一个新的 Todo 列表组件，使用 TypeScript 和 React
> 为这个函数添加错误处理
> 运行测试并修复失败的用例
```

### 使用 `@` 引用文件

在对话中引用特定文件：

```
> @src/utils/auth.ts 添加 token 刷新逻辑
> @src/components/Header.tsx 修复布局问题
```

### 多轮对话

Codex 会保持上下文，你可以进行多轮交互：

```
> 创建一个用户注册表单
> 添加表单验证
> 把提交按钮改为蓝色
> 添加单元测试
```

## 文件操作

### 读取文件

Codex 会自动读取你引用的文件，也可以主动要求：

```
> 请读取 package.json 并总结依赖项
```

### 创建文件

```
> 在 src/utils/ 目录下创建 formatDate.ts，包含日期格式化函数
```

### 编辑文件

```
> 将 index.js 中的 var 改为 const
> 重构 UserService 类，使用 async/await 替代 Promise 链
```

## 命令执行

Codex 可以在终端执行命令，例如：

```
> 运行 npm test
> 执行 git status
> 用 prettier 格式化所有文件
```

### 权限管理

Codex 的沙箱系统会根据项目信任级别决定执行权限：

| 信任级别 | 文件读取 | 文件写入 | 命令执行 |
|---------|---------|---------|---------|
| 受信任 | ✅ | ✅ | ✅ 自动 |
| 未受信任 | ✅ | ⚠️ 需审批 | ⚠️ 需审批 |

## 快捷键

| 快捷键 | 功能 |
|--------|------|
| `Enter` | 发送输入 |
| `Shift+Enter` | 换行 |
| `Ctrl+C` | 取消当前操作 |
| `Ctrl+D` | 退出会话 |
| `Esc` | 中断 AI 输出 |
| `↑` / `↓` | 浏览历史输入 |

## 使用技巧

### 1. 描述要具体

```
# ❌ 模糊
> 优化代码

# ✅ 具体
> 将 parseConfig 函数的嵌套 if-else 替换为策略模式，减少圈复杂度
```

### 2. 提供上下文

```
> 这个函数的时间复杂度是 O(n²)，当数据量达到 10 万时会很慢，请优化为 O(n log n)
```

### 3. 分步骤执行

复杂任务拆分为小步骤，更容易获得理想结果：

```
> 第一步：创建数据模型和类型定义
> 第二步：实现 API 客户端
> 第三步：创建 UI 组件
> 第四步：编写集成测试
```

### 4. 使用技能

通过 Skill 前缀触发特定工作流：

```
> 用 $brainstorming 功能探索新的 API 设计方案
> 使用 $security-scan 检查这个仓库
```

## 会话管理

### 查看会话历史

```bash
codex sessions list
```

### 执行特定操作

```bash

codex --help

# 查看版本
codex --version
```

# receiving-code-review

接收代码审查反馈技能，强调技术严谨性验证而非形式化服从。

## 核心原则

- 每条反馈都进行技术验证
- 不盲目同意所有建议
- 对有疑问的建议要求澄清

## 处理流程

1. **阅读审查反馈**：理解每个建议
2. **技术验证**：判断建议是否正确
3. **分类处理**：
   - ✅ 正确 → 实施修改
   - ❌ 不正确 → 礼貌解释原因
   - ⚠️ 不确定 → 进一步研究
4. **实施修改**：按验证后的建议进行修改
5. **回复说明**：对每个反馈给出回应

## 回复模板

```
感谢您的审查！
- [建议 1]：已按建议修改
- [建议 2]：经过验证，原实现是正确的，原因是...
- [建议 3]：不太确定，能否提供更多上下文？
```

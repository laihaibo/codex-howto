# frontend-testing-debugging

前端应用测试与调试技能，通过 Playwright 与本地 Web 应用交互，验证功能、调试 UI 问题。

## 触发条件

- 测试本地 Web 应用的功能
- 调试 UI 渲染或交互问题
- 检查控制台错误
- 验证响应式布局
- 截图比较和视觉回归测试

## 常用操作

```
> 启动 dev server 并测试登录流程
> 截图保存当前页面状态
> 检查控制台是否有报错
> 测试页面在移动端的响应式效果
> 对比修改前后的视觉差异
```

## 功能列表

| 功能 | 说明 |
|------|------|
| 页面导航 | 访问 URL、点击链接、表单提交 |
| 截图对比 | 全页面/元素级截图 |
| 控制台检查 | 获取 JS 错误和日志 |
| 交互模拟 | 点击、输入、拖拽等 |
| 响应式测试 | 模拟不同设备和视口 |

## 注意事项

- 确保 Dev Server 正在运行
- 使用 `webapp-testing` 技能进行系统化测试
- 视觉 QA 需在同一视口大小下对比

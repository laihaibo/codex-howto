# yeet

将本地更改发布到 GitHub 并创建草稿 PR 的技能。

## 触发条件

```
> 提交并推送这个分支创建 PR
> 把当前改动发到 GitHub
> 发布这些修改
```

## 工作流程

1. **确认范围**：检查当前 git 状态，确认要提交的文件
2. **创建提交**：分阶段编辑并有意图地创建提交
3. **推送分支**：推送到远程仓库
4. **创建草稿 PR**：通过 GitHub App 创建草稿 PR
5. **返回链接**：提供 PR 链接

## 使用示例

```
> 提交当前所有改动并推送到 origin，然后创建草稿 PR
```

## 提交规范

- 每个提交聚焦单一变更
- 信息清晰描述做了什么和为什么
- 遵循项目的提交风格

## 与 Codex 工具配合

使用 GitHub App 进行 PR 操作，在 connector 不足时回退到 `gh` 命令。

# test-driven-development

测试驱动开发技能，先编写测试用例再实现功能。

## TDD 循环

```
Red → Green → Refactor
写失败测试 → 实现功能通过测试 → 重构优化
```

## 使用示例

```
> 用 TDD 实现密码强度验证器
> $tdd 创建用户认证的 API 端点
```

## 工作流程

1. **编写失败测试**：描述功能期望行为
2. **运行测试确认失败**：Red 阶段
3. **编写最小实现**：让测试通过
4. **运行测试确认成功**：Green 阶段
5. **重构优化**：保持测试绿色

## 最佳实践

- 一次只写一个测试
- 测试要小而具体
- 生产代码只需满足测试要求
- 频繁运行测试
- 重构时保持测试不变

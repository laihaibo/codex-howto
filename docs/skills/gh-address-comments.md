# gh-address-comments

处理 GitHub PR 审查评论中的可操作反馈。

## 触发条件

```
> 处理 PR 上的审查评论
> 修复 reviewer 提出的问题
> 回答所有未解决的评论
```

## 工作流程

1. **获取评论**：通过 GraphQl 获取线程级评审反馈
2. **筛选可操作项**：过滤掉已解决和纯信息性评论
3. **逐个实现修复**：根据评论内容修改代码
4. **确认已解决**：标记评论为已解决

## 评论类型处理

| 评论类型 | 处理方式 |
|---------|---------|
| 代码修改建议 | 应用建议的修改 |
| 问题询问 | 回答并修改代码 |
| 风格偏好 | 按需调整 |
| LGTM/approve | 无需操作 |

## 最佳实践

- 对每个评论提供简洁的回复说明做了什么修改
- 不确定的评论标记为需要进一步讨论
- 完成后更新状态以便追踪

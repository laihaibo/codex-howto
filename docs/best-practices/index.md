# 最佳实践概述

本文档总结了使用 Codex CLI 的最佳实践，帮助你更高效地发挥这个 AI 编程助手的能力。

## 核心哲学

### 1. 具体胜于含糊

```
# ❌ 模糊
> 优化代码

# ✅ 具体  
> 将 parseConfig 函数的嵌套 if-else 替换为策略模式，减少圈复杂度
```

### 2. 上下文为王

提供足够的背景信息，让 AI 理你真正想要什么：

```
> 这个排序函数目前时间复杂度 O(n²)，当用户数据达到 10 万条时会卡顿，
> 请改为 O(n log n) 的实现，保持类型签名不变。
```

### 3. 渐进式交互

大任务拆分为小步骤，每步确认结果：

```
> 第一步：定义数据模型和类型
> 第二步：实现 API 客户端
> 第三步：创建 UI 组件
> 第四步：添加集成测试
```

### 4. 验证结果

不要假设 AI 生成的代码正确工作：

```
> 运行类型检查，确保没有类型错误
> 执行测试，确认所有用例通过
> 在浏览器中验证渲染效果
```

## 通用技巧

### 1. 善用技能系统

- 了解你已安装的技能和插件
- 在描述需求时让 Codex 自动匹配技能
- 使用 `$SkillName` 手动触发特定技能

### 2. 文件引用

在对话中使用 `@filename` 引用文件，帮助 Codex 定位上下文。

### 3. 反馈循环

如果结果不理想，提供具体反馈重新生成：

```
> 这个实现太复杂了，请简化
> 改用 React Hooks 而不是类组件
> 增加错误处理
```

### 4. 善用计划模式

复杂任务进入计划模式让 Codex 分解可行方案，避免盲目操作。

## 适用版本

本文档基于 **Codex CLI v0.142.5**。
